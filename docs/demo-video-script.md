# Demo Video Script

Target length: 3 minutes.

1. Problem, 0:00-0:25
   - Online hackathons and bounties look like a fast path to cash, but the hard part is choosing a reachable prize route under deadline pressure.
   - PrizePilot treats that as an autopilot workflow: it reads opportunities, ranks them, prepares artifacts, and keeps risky external actions in a review queue.

2. Portfolio triage, 0:25-0:55
   - Open the dashboard and show the ranked portfolio.
   - Explain that the agent scores prize amount, winner count, deadline pressure, evidence requirements, and blockers.
   - Call out why a smaller multi-winner feedback prize can be urgent while the Qwen route remains the higher-upside project route.

3. Qwen route, 0:55-1:25
   - Open the Qwen plan and show the Blog Post Award / Honorable Mention target.
   - Explain the Qwen-ready DashScope client: local deterministic planning is auditable, and a live Qwen refinement pass can improve narratives once a runtime key is provided.
   - Be explicit that the key is never stored in the repo or ledger.

4. Multi-agent workflow, 1:25-1:55
   - Walk through the five roles: Contest Analyst, Competitor Scout, Differentiation Strategist, Submission Coach, and Risk Reviewer.
   - Show how each role produces a concrete next action rather than generic advice.

5. Trust and control, 1:55-2:25
   - Show the approval queue for API keys, OAuth, cloud deployment, payout, tax, KYC, and public submission updates.
   - Show risk badges and evidence gaps: live Qwen check and live Alibaba Cloud endpoint are still open until verified.

6. Judge view, 2:25-2:50
   - Show the judge scorecard aligned to Qwen criteria: innovation, technical depth, problem value, and presentation.
   - Show `/api/plan` to demonstrate that the same plan is machine-readable.

7. Close, 2:50-3:00
   - PrizePilot is submitted with a public repo, Vimeo demo, and build journal.
   - The next improvement is stronger live sponsor evidence before judging, not overstated claims.
